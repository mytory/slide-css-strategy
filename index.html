시맨틱
콘텐츠의 성격을 반영하는 클래스명 모양 X
시맨틱은 CSS 스펙에 나옴. "… 작성자는 콘텐츠의 성격(nature)을 설명하는 [클래스 속성] 값을 사용하는 편이 좋다. 콘텐츠 모양이 어떻게 표시되길 바라는가를 설명하는 값을 사용하는 것은 좋지 않다." 계단에 따라 적용할 규칙을 덮어쓸 수 있다는 점때문에 불필요하게 많은 선택자가 사용된다.

계층적 선택자를 배워 봅시다

.issue { ... }
.issue h2 { ... }
#intro h1 { ... }
#intro .main { ... }
#intro .main h1 { ... }

한 화면에 담으려고 한 줄로 썼습니다.

이런 식의 예제를 심심찮게 볼 수 있다. 학습으로선 괜찮다.

저격형 선택자. 엄청나게 비효율적이다. 문서의 구조를 담으려는 노력도 보인다. 1. 비슷한 CSS 속성이 반복 2. 새 페이지와 요소를 추가할 때마다 CSS를 새로 작성하거나 편집. 3. 선택자 점수 경쟁 4. CSS 용량 증가 -> 성능에 악영향 5. 불필요한 코드 삭제에 번거롭다

클래스명은 내용을 설명해야 하는가?

CSS의 시맨틱 ≠ 내용의 시맨틱

by 니콜라스 갤러거

그런데, 클래스명이 왜 꼭 내용에서 도출돼야 하는가? 1. 검색엔진은 클래스명을 참조하지 않는다. (예외: 마이크로포맷) 2. 사람을 위해서? 내용을 반복할 뿐이다.
내용 시맨틱
검색엔진, 사람이 참조
→ HTML, Microdata
개발 시맨틱
개발자가 참조
→ CSS 클래스명
1. 내용의 시맨틱을 위해서는 마크업과 마이크로데이터를 신경써라. 검색엔진과 사람을 위한 것이다. 2. CSS는 개발자가 주 사용자다. 개발자를 위한 시맨틱을 신경써야 한다. 
<h2>뉴스</h2>
<ul class="news">
  ...
<h2>뉴스</h2>
<ul class="simple-list">
  ...
1. `.news`: 뉴스에만 사용할 수 있고, 정보는 내용에서 도출된다. 2. `.simple-list`: 비슷한 형태라면 어디서나 재사용할 수 있고, 어떤 목적으로 사용하는 클래스인지 개발자에게 정보를 준다.
